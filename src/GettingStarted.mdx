# 💻 Guia do Desenvolvedor: Como utilizar o Design System

Este guia foi criado para responder à pergunta: *"Como eu pego esses componentes bonitos e coloco no meu código?"*

O OneDocs Design System não é instalado via `npm install ds-onebox`. Ele segue a filosofia "copiar e colar" (baseada no Shadcn UI), onde você é dono do código. 

Este repositório (`DS-OneBox`) é a **Fonte da Verdade**. Sempre que um dev precisar de um componente, ele deve vir aqui, copiar o código e colar no seu próprio projeto.

---

## 1. Configurando seu Projeto Base

Antes de copiar os componentes, seu projeto (ex: Next.js ou Vite) precisa estar preparado:

1. **Tailwind CSS:** Certifique-se de que o Tailwind está instalado e configurado.
2. **Utilitários:** Copie o arquivo `src/lib/utils.ts` deste repositório para o seu (ele contém a função `cn` que mescla classes do Tailwind).
3. **Estilos Globais:** Copie as variáveis CSS (Cores e Dark Mode) do arquivo `src/index.css` deste repositório para o arquivo CSS global do seu projeto.
4. **Dependências Core:** Instale as bibliotecas base que quase todos os componentes usam:
   ```bash
   npm install class-variance-authority clsx tailwind-merge lucide-react @radix-ui/react-slot
   ```

---

## 2. Copiando Componentes (O Fluxo de Trabalho)

Sempre que precisar de um botão, tabela ou modal novo:

1. Abra o repositório principal do Design System no GitHub (ou clone-o na sua máquina).
2. Vá até a pasta `src/components/shadcn` (para primitivos) ou `src/components/blocks` (para padrões complexos).
3. Copie o arquivo `.tsx` do componente desejado.
4. Cole dentro da pasta equivalente no seu projeto (ex: `src/components/shadcn/Button.tsx`).
5. **Atenção às dependências:** Se você copiar um componente complexo (como o `Dialog` ou `Select`), veja os imports no topo do arquivo. Você pode precisar instalar os pacotes do Radix UI referentes a ele:
   ```bash
   npm install @radix-ui/react-dialog
   ```

---

## 3. Importando e Utilizando no seu Código

Para utilizar qualquer componente, basta importá-lo do caminho local do seu projeto. Evite caminhos relativos longos (\`../../../\`) configurando o alias \`@\` no seu \`tsconfig.json\`.

\`\`\`tsx
// ✅ Faça assim (usando alias):
import { Button } from "@/components/shadcn/Button"
import { Card, CardHeader, CardTitle, CardContent } from "@/components/shadcn/Card"

// ❌ Evite assim:
import { Button } from "../../../components/shadcn/Button"
\`\`\`

---

## 4. Utilizando as Variantes (Variants)

A maioria dos nossos componentes primitivos usa a biblioteca \`class-variance-authority\` (cva) por baixo dos panos. Isso significa que eles possuem "modos" pré-programados de cores e tamanhos.

Você não precisa recriar estilos. Basta passar as props \`variant\` e \`size\`:

\`\`\`tsx
export function MinhaTela() {
  return (
    <div className="flex gap-4">
      {/* Botão Padrão (Azul) */}
      <Button>Salvar</Button>

      {/* Botão Secundário e Pequeno */}
      <Button variant="secondary" size="sm">Cancelar</Button>

      {/* Botão de Erro e Grande */}
      <Button variant="destructive" size="lg">Excluir Conta</Button>
    </div>
  )
}
\`\`\`

> 💡 **Como saber quais variants existem?**
> Navegue até o componente aqui no Storybook. Role até a seção **"Controls"** no final da página. Lá você verá um *dropdown* com todas as opções de \`variant\` e \`size\` disponíveis para testar ao vivo!

---

## 5. Consultando o Código no Storybook

Você não precisa adivinhar como montar componentes complexos (como um Modal, uma Tabela ou um Acordeão). 

Em **qualquer componente** aqui no Storybook:
1. Vá para a página do componente.
2. Olhe o bloco de exemplo renderizado.
3. No canto inferior direito desse bloco, clique no botão **"Show Code"**.
4. Copie a estrutura exata do React (JSX) e cole no seu arquivo!

---

## 6. Sobrescrevendo Estilos (className)

Nosso sistema é 100% construído com **Tailwind CSS**. 
Todos os componentes expõem a propriedade \`className\`. Isso significa que se um componente não se encaixar perfeitamente no seu layout, você pode passar classes do Tailwind para forçar margens, larguras ou comportamentos específicos.

O sistema usa o \`tailwind-merge\` por trás das cortinas, então a sua classe **sempre terá prioridade** sobre a classe padrão do componente.

\`\`\`tsx
import { Input } from "@/components/shadcn/Input"

export function MeuFormulario() {
  return (
    // O Input já tem seus estilos de borda e foco.
    // Estamos apenas adicionando uma margem superior (mt-4) e forçando largura (w-full).
    <Input 
      placeholder="Digite seu nome..." 
      className="mt-4 w-full md:w-[300px]" 
    />
  )
}
\`\`\`

## 7. Lidando com Ícones

Nós usamos a biblioteca \`lucide-react\`. Para adicionar um ícone ao lado de um texto (como em botões), importe-o diretamente e ajuste o tamanho com as classes \`w-4 h-4\`:

\`\`\`tsx
import { Button } from "@/components/shadcn/Button"
import { Mail, ArrowRight } from "lucide-react"

export function BotoesComIcone() {
  return (
    <div className="flex gap-4">
      <Button>
        <Mail className="mr-2 w-4 h-4" /> Entrar com Email
      </Button>

      <Button variant="outline">
        Avançar <ArrowRight className="ml-2 w-4 h-4" />
      </Button>
    </div>
  )
}
\`\`\`
