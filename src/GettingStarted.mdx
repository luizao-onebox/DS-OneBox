# 💻 Guia do Desenvolvedor: Como utilizar o Design System

Este guia foi criado para responder à pergunta: *"Como eu pego esses componentes bonitos e coloco no meu código?"*

O OneDocs Design System é empacotado e distribuído como uma biblioteca NPM privada (`ds-onebox`). Este repositório (DS-OneBox) é a única fonte da verdade, e **os desenvolvedores devem consumir o pacote gerado, sem alterar o código dos componentes diretamente em suas aplicações.**

Essa abordagem garante que todos os projetos (Admin, Portais, etc.) fiquem consistentes e atualizados automaticamente quando o Design System evoluir.

---

## 1. Instalação

No seu projeto React/Next.js/Vite, instale o pacote do Design System e as dependências pares necessárias:

```bash
# Como o pacote é privado, instale diretamente do repositório do GitHub
# O comando aponta para a branch "storybook" onde o código está gerado
npm install git+https://github.com/luizao-onebox/DS-OneBox.git#storybook

# Instale os ícones que são usados em conjunto com o DS
npm install lucide-react
```

---

## 2. Configurando o Tema (Tailwind CSS)

Para que os componentes do pacote tenham a mesma cara do Storybook, seu projeto precisa herdar as variáveis de cor e as configurações do Tailwind.

> ⚠️ **Aviso Importante: Limpando a Casa!**
> Se você criou seu projeto usando ferramentas como `create-vite`, elas injetam um arquivo `index.css` ou `App.css` padrão que possui regras globais (ex: `#root { display: flex; flex-direction: column; text-align: center; }`). Essas regras podem quebrar completamente o layout dos nossos componentes (como o `Sidebar`). 
> **A regra de ouro é: apague todo o CSS boilerplate do seu framework antes de começar.** O seu CSS global deve conter idealmente apenas as diretivas do Tailwind (`@tailwind base; @tailwind components; @tailwind utilities;`) e a importação do nosso Design System.

### Passo 2.1: Importar o CSS Global
No seu arquivo principal (ex: `main.tsx`, `App.tsx` ou `layout.tsx` no Next.js), importe o CSS compilado do pacote logo no topo:

```tsx
import 'ds-onebox/style.css'
```

### Passo 2.2: Configurar o Tailwind do seu App
Seu projeto precisa ler as classes do Tailwind de dentro do `node_modules` do design system e herdar as configurações exatas da empresa (breakpoints, cores, espaçamentos). Para isso, use o nosso preset no seu `tailwind.config.js` (ou `tailwind.config.ts`):

```js
import dsOneBoxPreset from "ds-onebox/tailwind-preset"

/** @type {import('tailwindcss').Config} */
export default {
  presets: [dsOneBoxPreset], // Puxa todas as regras visuais da empresa
  content: [
    "./src/**/*.{js,ts,jsx,tsx}",
    // Escaneia os componentes compilados do Design System
    "./node_modules/ds-onebox/dist/**/*.js"
  ],
}
```

> **Atenção:** Se o seu projeto usar CommonJS (sem `type: "module"`), você pode usar `require("ds-onebox/tailwind-preset")` caso o Node permita, mas a biblioteca moderna exporta via ES Modules.

## 3. Primitivos de Layout (Layout Primitives)

Para que você não precise "adivinhar" ou configurar regras do Tailwind do zero para montar suas telas, o Design System exporta componentes estruturais que já embutem as regras de espaçamento (tokens), colunas e breakpoints da empresa.

Exemplos de componentes disponíveis:
- `<Container>`: Centraliza o conteúdo e define a largura máxima (max-width) padrão das telas.
- `<Grid>`: Define o sistema de colunas e ajusta automaticamente as margens em telas menores.
- `<Flex>`: Facilita o alinhamento de itens (horizontal ou vertical) com espaçamentos padronizados (`gap`).
- `<Section>`: Cria os blocos de conteúdo com espaçamento correto (`padding`) em cima e embaixo.

**Como usar na prática:**

```tsx
import { Container, Grid, Card } from "ds-onebox"

export function MinhaPagina() {
  return (
    <Container>
      {/* Grade responsiva que muda de 1 para 2 e depois para 4 colunas dependendo da tela */}
      <Grid columns={{ sm: 1, md: 2, lg: 4 }} gap="md">
        <Card>Card 1</Card>
        <Card>Card 2</Card>
        <Card>Card 3</Card>
        <Card>Card 4</Card>
      </Grid>
    </Container>
  )
}
```

## 4. Importando e Utilizando no seu Código

Para utilizar qualquer componente, você os importa diretamente da biblioteca `ds-onebox` via NPM.

```tsx
// ✅ Faça assim (importando da lib NPM):
import { Button, Card, CardHeader, CardTitle, CardContent } from "ds-onebox"

export function MinhaTela() {
  return (

      </CardHeader>
      <CardContent>
        <Button variant="default">Acessar</Button>
      </CardContent>
    </Card>
  )
}
```

## 5. Utilizando as Variantes (Variants)

A maioria dos nossos componentes primitivos usa a biblioteca `class-variance-authority` (cva) por baixo dos panos. Isso significa que eles possuem "modos" pré-programados de cores e tamanhos.

Você não precisa recriar estilos. Basta passar as props `variant` e `size`:

```tsx
export function MinhaTela() {
  return (
    <div className="flex gap-4">
      {/* Botão Padrão (Azul) */}
      <Button>Salvar</Button>

      {/* Botão Secundário e Pequeno */}
      <Button variant="secondary" size="sm">Cancelar</Button>

      {/* Botão de Erro e Grande */}
      <Button variant="destructive" size="lg">Excluir Conta</Button>
    </div>
  )
}
```

> 💡 **Como saber quais variants existem?**
> Navegue até o componente aqui no Storybook. Role até a seção **"Controls"** no final da página. Lá você verá um *dropdown* com todas as opções de `variant` e `size` disponíveis para testar ao vivo!

## 7. Sobrescrevendo Estilos (className)

Nosso sistema é 100% construído com **Tailwind CSS**. 
Todos os componentes expõem a propriedade `className`. Isso significa que se um componente não se encaixar perfeitamente no seu layout, você pode passar classes do Tailwind para forçar margens, larguras ou comportamentos específicos.

O sistema usa o `tailwind-merge` por trás das cortinas, então a sua classe **sempre terá prioridade** sobre a classe padrão do componente.

```tsx
import { Input } from "ds-onebox"

export function MeuFormulario() {
  return (
    // O Input já tem seus estilos de borda e foco.
    // Estamos apenas adicionando uma margem superior (mt-4) e forçando largura (w-full).
    <Input 
      placeholder="Digite seu nome..." 
      className="mt-4 w-full md:w-[300px]" 
    />
  )
}
```

## 8. Lidando com Ícones

Nós usamos a biblioteca `lucide-react`. Para adicionar um ícone ao lado de um texto (como em botões), importe-o diretamente e ajuste o tamanho com as classes `w-4 h-4`:

```tsx
import { Button } from "ds-onebox"
import { Mail, ArrowRight } from "lucide-react"

export function BotoesComIcone() {
  return (
    <div className="flex gap-4">
      <Button>
        <Mail className="mr-2 w-4 h-4" /> Entrar com Email
      </Button>

      <Button variant="outline">
        Avançar <ArrowRight className="ml-2 w-4 h-4" />
      </Button>
    </div>
  )
}
```

## 9. Temas White-Label (Multi-Marca)

O Design System é **nativamente preparado para multi-marca**. Cada cliente/edição pode ter seu próprio tema visual sem alterar o código dos componentes.

### Como funciona

O DS usa **CSS Custom Properties (tokens)** para todas as cores, tipografia e espaçamentos. Cada tema é um arquivo CSS que sobrescreve esses tokens.

```
Seu Projeto/
├── node_modules/
│   └── ds-onebox/           ← Design System (não alterar)
├── themes/
│   └── tema-minha-marca.css ← Seu tema customizado
└── src/
    └── App.tsx
```

### Criando um novo tema

**1.** Crie um arquivo CSS em `themes/`:

```css
/* themes/tema-minha-marca.css */

/* Sobrescreve os tokens que deseja customizar */
:root[data-theme="minha-marca"] {
  /* Cor primária da marca */
  --primary: 262 83% 58%;
  --primary-foreground: 0 0% 100%;

  /* Fonte da marca */
  --font-sans: "Nunito Sans", sans-serif;

  /* Border radius mais arredondado */
  --radius: 0.75rem;
}

/* Dark mode do tema */
:root[data-theme="minha-marca"] .dark,
:root[data-theme="minha-marca"].dark {
  --primary: 262 83% 68%;
  --primary-foreground: 0 0% 100%;
}
```

**2.** No seu projeto, importe o CSS na ordem correta:

```tsx
// main.tsx ou App.tsx
import 'ds-onebox/style.css'
import './themes/tema-minha-marca.css'
```

**3.** Aplique o tema no elemento HTML:

```tsx
// No seu componente raiz (App.tsx ou layout.tsx)
<html lang="pt-BR" data-theme="minha-marca">
  <body>
    {/* sua aplicação */}
  </body>
</html>
```

### Boas práticas para temas

| Regra | Por quê |
|-------|---------|
| Sobrescreva apenas tokens semânticos | `--primary`, `--success`, `--radius` etc. |
| Nunca use valores hardcoded | `bg-blue-500`, `text-red-600` |
| Sempre defina light E dark | O usuário pode trocar o modo |
| Teste contraste WCAG | Tokens do DS garantem 4.5:1, mudanças podem quebrar |

### Tokens disponíveis

Consulte o arquivo `tokens-manifest.css` no diretório `themes/` para ver todos os 100+ tokens que podem ser customizados.

## 10. Gerando Telas com IA

O Design System foi pensado para **geração de telas por Inteligência Artificial**. Para garantir que a IA use apenas tokens válidos:

1. **Forneça o `tokens-manifest.css`** como referência para a IA
2. **Instrua a IA** a usar apenas tokens semânticos (`bg-primary`, `text-primary-foreground`)
3. **Nunca usar** valores hardcoded como `bg-blue-500`

Exemplo de prompt para IA:

> "Use o DS-OneBox. Crie um dashboard com Cards, Badges e Buttons. Use exclusivamente tokens semânticos: `bg-primary`, `text-primary-foreground`, `bg-success-100`, `text-success-800`. Não use cores hardcoded."

## 11. Solução de Problemas

### Conflito de nomes com Recharts

Tanto o `ds-onebox` quanto o `recharts` exportam um componente chamado `Tooltip`. Quando usar ambos no mesmo arquivo, o segundo import sobrescreve o primeiro silenciosamente.

**Solução:** Use alias no import do Recharts:

```tsx
// ✅ Correto — alias para evitar conflito
import { Tooltip, TooltipTrigger, TooltipContent } from 'ds-onebox'
import { Tooltip as RechartsTooltip, Legend } from 'recharts'

// Uso
<RechartsTooltip formatter={(v) => formatCurrency(v)} />
<Tooltip><TooltipTrigger>...</TooltipTrigger></Tooltip>
```

### Grid e MetricGrid com prop `columns`

A prop `columns` aceita **números inteiros**, não objetos:

```tsx
// ✅ Correto — número
<Grid columns={4} gap="md" />
<MetricGrid columns="4" gap="md" />

// ❌ Errado — objeto (não funciona)
<Grid columns={{ xs: 1, sm: 2, lg: 4 }} />
<MetricGrid columns={{ xs: 1, sm: 2, lg: 4 }} />
```

Para layouts responsivos customizados, use `className`:

```tsx
<Grid columns={2} gap="md" className="lg:grid-cols-3" />
```

### Props do DataTable

O DataTable usa `TanStack Table` internamente. Ações (botões, links) devem ser definidas como **colunas explícitas** dentro do array de `columns`:

```tsx
// ✅ Correto — coluna de ações explícita
{
  id: 'acoes',
  cell: ({ row }) => (
    <Button variant="ghost" size="icon" onClick={() => handleView(row.original)}>
      <Eye className="h-4 w-4" />
    </Button>
  ),
}

// ❌ Errado — onRowClick não existe no DataTable
<DataTable columns={columns} data={data} onRowClick={handleView} />
```

## 12. Compatibilidade com Tailwind CSS

O DS-OneBox foi construído com **Tailwind CSS v4**. Se você está enfrentando problemas de compatibilidade, esta seção explica como resolver.

### Arquitetura Atual

| Arquivo | Abordagem | Versão |
|---------|-----------|--------|
| `src/index.css` | `@import "tailwindcss"` + `@theme {}` | v4 |
| `tailwind-preset.js` | `theme.extend` + `plugins` | v3 |
| `tailwind.config.js` | Importa o preset | v3 + v4 hybrid |

### Problemas Comuns

#### Problema: `@layer base` conflitando

**Sintoma:** Estilos do DS-Onebox sobrepõem os do seu projeto, ou vice-versa.

**Solução:** Não importe o CSS compilado do ds-onebox. Use apenas o preset:

```tsx
// ❌ Errado — importa CSS que pode conflitar
import 'ds-onebox/style.css'

// ✅ Correto — usa apenas o preset
// O preset define classes Tailwind, não estilos CSS globais
```

#### Problema: `tailwind-preset.js` não funciona

**Sintoma:** Classes como `bg-primary`, `text-foreground` não existem.

**Causa:** O preset é um config válido para v3, mas precisa ser referenciado corretamente.

**Solução:** No seu `tailwind.config.js` (v3 ou v4):

```js
// tailwind.config.js (v3)
const dsOneboxPreset = require('ds-onebox/tailwind-preset')

module.exports = {
  presets: [dsOneboxPreset],
  // seu config...
}
```

```js
// tailwind.config.js (v4)
import dsOneboxPreset from 'ds-onebox/tailwind-preset'

export default {
  presets: [dsOneboxPreset],
}
```

#### Problema: Variáveis CSS não definidas

**Sintoma:** Cores funcionam mas espaçamento/typografia não.

**Causa:** As variáveis CSS (ex: `--radius`, `--font-sans`) precisam ser definidas no CSS do projeto.

**Solução:** Copie as variáveis do `src/themes/tokens-manifest.css` para o seu CSS principal:

```css
/* No seu index.css */
@import 'ds-onebox/tokens.css';

:root {
  --radius: 0.5rem;
  --font-sans: 'Inter', ui-sans-serif, system-ui, sans-serif;
}
```

### Para Consumidores v3

Se seu projeto usa Tailwind v3:

```bash
npm install ds-onebox@^1.0.0 tailwindcss@^3
```

```js
// tailwind.config.cjs
const dsOneboxPreset = require('ds-onebox/tailwind-preset')

module.exports = {
  content: ['./src/**/*.{js,ts,jsx,tsx}'],
  presets: [dsOneboxPreset],
  theme: {
    // sobrescreva tokens aqui se precisar
  },
}
```

### Para Consumidores v4

Se seu projeto usa Tailwind v4:

```bash
npm install ds-onebox@^1.0.0 tailwindcss@^4
```

```js
// tailwind.config.js
import dsOneboxPreset from 'ds-onebox/tailwind-preset'

export default {
  presets: [dsOneboxPreset],
}
```

```css
/* seu index.css */
@import 'ds-onebox/tokens.css';
@import 'tailwindcss';
```

### Checklist de Verificação

- [ ] `tailwindcss` está instalado (v3 ou v4)
- [ ] `tailwind.config.js` importa `ds-onebox/tailwind-preset`
- [ ] Tokens CSS ( `--primary`, `--radius`, etc.) estão no CSS do projeto
- [ ] Não há importação de `ds-onebox/style.css` (use só o preset)
- [ ] Se usar v3: `@layer base` no CSS do DS não entra em conflito
